#+TITLE: riffl Roadmap
#+DATE: 2026-03-13
#+DESCRIPTION: Development roadmap. Floor: ProTracker parity. Ceiling: Renoise-level power and fun.

* Goals
  Minimum target: feature parity with ProTracker (Amiga MOD tracker).
  Real target: the power and feel of Renoise — deep instrument system, flexible
  pattern editing, great DSP, live recording, and a scripting DSL that no
  commercial tracker has. Chip emulation (Furnace-style) is a potential future
  direction but not a priority.

  See VISION.md for the long-horizon plan (GUI, plugin hosting, monetisation).

* Milestone 1 — ProTracker Parity
  :PROPERTIES:
  :STATUS: in-progress
  :END:

** Effect Commands
   ProTracker has 10 top-level effects + 14 extended (Exy) sub-effects.
   We implement 9 of the top-level ones. Missing:

*** TODO 5xy — Tone Portamento + Volume Slide
    Combine effect 3 (portamento-to-note) and A (volume slide) in one column.
    Param: same nibble split as Axy (upper = slide up, lower = slide down).

*** TODO 6xy — Vibrato + Volume Slide
    Combine effect 4 (vibrato) and A (volume slide).
    Continues vibrato from previous row while sliding volume.

*** TODO 7xy — Tremolo
    Periodic volume modulation (analogous to vibrato but on amplitude).
    Upper nibble = speed, lower = depth. Same waveform table as vibrato.
    State needed: tremolo_phase, tremolo_speed, tremolo_depth, tremolo_waveform.

*** DONE 9xx — Sample Offset
    Start sample playback at offset (xx * 256) frames into the sample.
    Implemented: Voice.with_position() sets initial position at note-on.

*** TODO Exy — Extended Effects (command 0xE, sub-command in upper param nibble)
    The E-effects share a single command slot; sub-command is param >> 4.

**** TODO E1x — Fine Portamento Up
     Like 1xx but applied once per row (not per tick). Param = semitones * 16.

**** TODO E2x — Fine Portamento Down
     Like 2xx but applied once per row.

**** TODO E3x — Glissando Control
     Toggle whether tone portamento (3xx) snaps to nearest semitone.
     E30 = smooth, E31 = glissando (semitone steps).

**** TODO E4x — Set Vibrato Waveform
     0=sine (default), 1=ramp-down, 2=square, 3=random.
     Waveform table used by 4xy and 6xy.

**** TODO E5x — Set Finetune
     Override instrument finetune for the current channel on this row.
     Param range 0–F maps to -8..+7 (ProTracker convention).

**** TODO E6x — Pattern Loop
     E60 = set loop start point; E6x (x>0) = loop x times back to start.
     State needed per-channel: loop_start_row, loop_count.

**** TODO E7x — Set Tremolo Waveform
     Same waveform options as E4x, applies to 7xy tremolo.

**** TODO E9x — Retrigger Note
     Re-trigger the current sample every x ticks within the row.

**** TODO EAx — Fine Volume Slide Up
     Like Ax0 but applied once per row.

**** TODO EBx — Fine Volume Slide Down
     Like A0x but applied once per row.

**** TODO ECx — Note Cut
     Set volume to 0 after x ticks within the row (sample keeps playing silently).

**** TODO EDx — Note Delay
     Delay note trigger by x ticks within the row.

**** TODO EEx — Pattern Delay
     Delay advancing to the next row by x extra row-lengths.

* Milestone 2 — Renoise-Level Instrument System
  :PROPERTIES:
  :STATUS: planned
  :END:

  This is what separates a toy tracker from a real composition tool.

** TODO Instrument Envelopes
   Per-instrument ADSR + LFO for volume, pitch, and panning.
   Applied on top of effect commands (commands take priority, envelope is default).
   - Volume envelope: shapes amplitude over note duration
   - Pitch envelope: automatic pitch glide/vibrato without effect columns
   - Panning envelope: auto-pan effects

** TODO Multi-Sample Instruments (Keyzone Mapping)
   One instrument can map different samples to different key ranges and/or
   velocity layers. E.g.: low notes use a bass sample, high notes use a
   different recording.
   Data model: =Instrument= gets =keyzones: Vec<Keyzone>=, each with
   =note_range=, =velocity_range=, =sample_index=.

** TODO Sample Slicing
   Mark slice points on a sample; each slice is addressable as a separate note
   in the pattern. Beat-chopping workflow. Essential for drum programming.

** TODO Per-Note Extra Columns (Renoise-style)
   Alongside Note + Instrument + Effect(s), add optional columns:
   - Volume column (00–FF, overrides instrument default)
   - Panning column (00–FF, L–R)
   - Delay column (00–FF ticks, delays note trigger within the row)
   These are per-cell, not global effects.

** TODO Phrases (Sub-Patterns inside Instruments)
   An instrument can contain mini-patterns ("phrases") that play back when the
   instrument is triggered. Like Renoise phrases. Enables complex rhythms
   triggered by a single note.

** TODO Sample Editor View
   Waveform display with loop point handles (set via cursor keys).
   Zoom in/out. Mark loop start/end. Normalize, reverse, fade in/out operations.

** TODO More Channels
   Support up to 64 channels (like Renoise). UI adapts with horizontal scroll.
   Current Vec<> is unbounded; the default and new-project dialog need updating.

** TODO Configurable Pattern Length
   Not locked to 64 rows. Per-pattern row count (16–512), like Renoise.
   Transport must handle variable-length patterns in the arrangement.

** TODO Undo/Redo Beyond Pattern Snapshots
   Global undo stack covering instrument edits, arrangement changes, etc.
   (Pattern-level undo already exists in Editor.)

* Milestone 2.5 — Pattern Editor & Transport Essentials
  :PROPERTIES:
  :STATUS: in-progress
  :END:

  These are blocking for real composition use. Should be done before
  the Renoise-level instrument system.

** DONE Better Keybindings for Entering Notes
FT2/IT piano keyboard layout: =a=C =w=C# =s=D =e=D# =d=E =f=F =t=F# =g=G =y=G# =h=A =u=A# =j=B =k=C+1oct

** DONE r — Replace-Once Mode
   Like vim's =r=: press =r=, type one note, cursor stays, mode stays Normal.
   No persistent insert, no step advance. Useful for fixing individual cells.
   Implemented: commit e813a24c.

** DONE Play From Cursor
   =Enter= (while stopped) starts playback from the current cursor row, not row 0.
   Decouple play-start position from the edit cursor.
   Implemented: =transport.play_from(row)= + =App::play_from_cursor()= + =Enter= in PatternEditor (stopped). Commit ea27f43c.

** DONE Loop Region
   Set loop start/end rows (=Alt+[= / =Alt+]=).
   Visual markers in the pattern row gutter: =[= start, =]= end, =¦= interior rows.
   Transport wraps at loop end back to loop start during playback.
   Toggle active with =Ctrl+Shift+L=. Footer shows =LOP HH-HH= pill when set.

** DONE Follow Mode
   Toggle =f= in Normal mode: edit cursor chases the playhead during playback.
   Without this, you can't see what's playing while it plays.
   Implemented: =follow_mode= field on App, wired in =update()= on AdvanceResult::Row.
   =FOL= pill shown in footer when active.

** DONE Transpose Selection
   Shift selected notes by semitone (=Shift+Up= / =Shift+Down=) or octave
   (=Ctrl+Shift+Up= / =Ctrl+Shift+Down=). Works on visual selection or current cell.
   Core YTPMV workflow — remapping chops to pitches constantly.
   Implemented: =transpose_selection(semitones)= on =Editor=. Normal and Visual
   modes both handled. Five unit tests covering single-cell, selection, empty
   cells, out-of-range clamping, and undo.

** DONE Note Cut (^^^) Distinct from Note Off (===)
   Add =NoteEvent::Cut= variant. Displays as =^^^=. Hard-silences the voice
   immediately (no envelope release). Currently =NoteEvent::Off= (===) does
   the same thing because there are no envelopes yet, but the distinction
   will matter once envelopes land.
   Also wire =ECx= extended effect (tick-delayed cut) in the mixer render.
   Key to enter note-cut: =Del= in Insert mode (=Backspace= still deletes cell).

** DONE Note Off Is Not Discoverable
   Note-off is currently bound to =`= (backtick) in Insert mode. This is
   not documented anywhere in the UI and differs from every tracker convention
   (FT2 uses =1=, IT/Renoise use =~= or a dedicated key).
   Fix: rebind to =~= or =1=, add to help screen and `:tutor`.

** DONE Per-Pattern Row Count
   Not locked to the global default. Each =Pattern= stores its own row count
   (16–512). Transport handles variable-length patterns in the arrangement.
   Essential for mixing short fills with long sections.
   =Ctrl+P= opens inline length prompt; =Len:N= shown in footer.

** DONE Draw Mode
   Hold =d= + note key to repeat-fill rows while held, or toggle draw mode
   where cursor auto-advances and repeats the last entered note.
   Speeds up drum pattern entry dramatically.
   Implemented: =draw_mode: bool= + =draw_note: Option<NoteEvent>= on App.
   =D= in Insert mode toggles draw mode. Cursor movement (j/down) when active
   auto-repeats last placed note. DRW pill shown in footer. Commit 57e92eee.

** DONE Note Interpolation
   Select a range → =I= to linearly interpolate volume or pitch between the
   first and last non-empty cell. Smooth automation without manual hex entry.
   Implemented: =interpolate_selection()= on =Editor=, wired to =I= in Visual mode.

** DONE BPM Inline Edit
   BPM is visible in the status bar but only editable via =:bpm N= command.
   Add =T= or =Ctrl+B= to open a quick BPM prompt.
   Bonus: tap-tempo (tap a key N times to set BPM from the interval).
   Implemented: =Ctrl+B= opens inline BPM prompt (pre-populated with current BPM,
   Enter applies, Esc cancels). =t= in Normal mode does tap-tempo — records tap
   timestamps, averages intervals over last 3 seconds, applies BPM after ≥2 taps.

** DONE :tutor — Interactive Tutorial / Cheat Sheet
   =:tutor= opens a navigable in-app reference covering:
   - All keybindings (Normal, Insert, Visual modes)
   - How to enter notes, note-off, note-cut, effects
   - How to use the sample browser (including the non-obvious Space/Enter split)
   - Transport controls and playback shortcuts
   - Effect command reference (0xy, 9xx, Exy table)
   Implemented as a full-screen scrollable overlay. j/k scroll, q/Esc close.
   Piano keyboard layout and all effects documented inline.

** TODO Sample Browser Overhaul
   Current browser only reads =sample_dirs= from config
   (default: =~/.config/riffl/samples=). Needed:
   - [DONE] =./samples/= relative to the open project file (auto-added if exists)
   - [DONE] Filesystem root escape — navigate to =/= or =~= and browse freely;
     =h= navigates up past configured roots; =~= jumps back to the roots list
   - [DONE] Configured roots remain as "pinned" (★) entries in the root list
   - [DONE] Favourites / bookmarks: =b= toggles bookmark on selected dir, persists
     to =config.toml=, bookmarked dirs shown at top of root list with ♥ marker
   Waveform panel: show waveform for selected file (lazy-load on selection).
   During Space preview: live playback cursor through waveform, time display.
   Preview toggle: Space = play/stop (currently Space always restarts).
   Scrub preview start: Left/Right while previewing to set start offset.

* Milestone 7 — Chip Sound Emulation (Furnace-style, optional future)
  :PROPERTIES:
  :STATUS: parked
  :END:

  Not a priority, but the architecture should not preclude it.
  Would require a =ChipVoice= variant alongside =SampleVoice= and =SynthVoice=.
  Candidates: YM2612 (Genesis FM), SN76489 (Game Boy/SMS), NES APU, SID (C64).
  Each chip has its own effect command vocabulary.
